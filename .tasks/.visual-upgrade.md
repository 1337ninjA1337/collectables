# Visual upgrade — Mixed direction

Implement a coherent visual upgrade across the app while leaving business
logic, data, contexts, and routing **untouched**. The direction below
(**Mixed**) combines the best parts of three explorations into one system.
Static design mocks live at `docs/visual-upgrade/Visual Upgrade Directions.html`
— open it in a browser to see the target.

State your direction choice in the commit message for PR #1.

---

## The Mixed direction in one paragraph

Keep the warm brown / amber palette exactly as-is. Soften the geometry
(radius 28→32 on cards and hero, gutter 20→24 on screen padding, soft
6/18 shadows on cards). Swap the display font from Syne to a system
serif chain (Iowan Old Style / Charter / Georgia) — adds editorial
character with zero new dependencies. Add a dark-mode token map driven
by `useColorScheme()` — no settings UI, just respect the OS preference.
Item detail gets an edge-to-edge hero photo. That's the whole change.

---

## Hard constraints (re-stated for clarity)

- **No new npm packages.** `Platform.select`, `useColorScheme`, and the
  expo-font setup already in place cover everything.
- **No new fonts beyond what's already loaded.** Syne and DM Sans are
  already loaded via `useFonts()` in `app/_layout.tsx`. Adding a system
  serif chain is allowed because it resolves to OS-bundled fonts (no
  asset). The Syne TTFs stay in `assets/fonts/` — just stop referencing
  them from display titles (see fonts section below for what stays).
- **No inline hex.** New colors → new tokens in `lib/design-tokens.ts`.
  `npm run lint:hex` must pass; `__tests__/check-inline-hex.test.ts` runs
  it as part of `npm test`.
- **No new strings outside i18n.** This upgrade introduces zero new copy,
  so the six language files in `lib/i18n-context.tsx` are left alone.
- **No logic changes.** No new `useState` outside the new theme hook, no
  new props on functional components beyond style-related ones, no new
  context reads. If a redesign needs new data, stop and flag it in the
  PR body.
- **Type-check, lint, and build must pass**: `npx tsc --noEmit && npm run
  lint:hex && npm run build`.
- **Tests must pass**: `npm test`. Mirror the structural test pattern
  from `__tests__/check-inline-hex.test.ts` for any token additions.
- **CLAUDE.md rules**: never run `npm start` / `web` / `ios` / `android`;
  push to **main** only; commit message = the PR's task name (no body);
  after pushing remove this `.tasks/.visual-upgrade.md` file from the
  repo as the final PR's last step. No credentials in commits.
- **No DB migrations.** No SQL. No `MANUAL-TASKS.md` additions.

---

## Out of scope

Do not redesign or touch:

- Anything under `lib/*-context.tsx`, `lib/supabase*.ts`,
  `lib/storage-keys.ts`, `lib/*-helpers.ts` — pure data / wiring.
- `__tests__/**` other than adding new token / theme tests in the shape
  of `check-inline-hex.test.ts`.
- The bottom-nav tab order, icon set, badge behavior, or modal create
  sheet — visual polish only inside the existing structure.

## PR 3 · `theme-hook` — `useAppTheme()`

Create `components/use-app-theme.ts`. This is a **hook**, not a context.
It reads `useColorScheme()` from `react-native` and returns a mapping of
semantic theme slots to existing design tokens. No new state, no new
context, no new logic.

```ts
import { useColorScheme } from "react-native";

import {
  AMBER_ACCENT,
  AMBER_LIGHT,
  AMBER_SOFT,
  BORDER,
  BORDER_2,
  CARD_BG,
  CARD_BG_2,
  CARD_BG_9,
  HERO_DARK,
  HERO_DARK_2,
  HERO_DARK_4,
  HERO_DARK_5,
  MUTED,
  MUTED_14,
  MUTED_19,
  MUTED_20,
  PAGE_BG_2,
  TEXT_DARK,
  TEXT_DARK_5,
  TEXT_ON_DARK,
  TEXT_ON_DARK_SOFT,
} from "@/lib/design-tokens";

export type AppTheme = {
  isDark: boolean;
  page: string;
  card: string;
  cardElevated: string;
  border: string;
  borderSoft: string;
  hero: string;
  text: string;
  textOnDark: string;
  textSoft: string;
  muted: string;
  meta: string;
  bannerBg: string;
  navBg: string;
  navIconActive: string;
  navIconInactive: string;
};

const LIGHT: AppTheme = {
  isDark: false,
  page: PAGE_BG_2,
  card: CARD_BG,
  cardElevated: CARD_BG_2,
  border: BORDER,
  borderSoft: BORDER_2,
  hero: HERO_DARK,
  text: TEXT_DARK,
  textOnDark: TEXT_ON_DARK,
  textSoft: TEXT_DARK_5,
  muted: MUTED_20,
  meta: MUTED,
  bannerBg: CARD_BG_9,
  navBg: CARD_BG_2,
  navIconActive: HERO_DARK,
  navIconInactive: MUTED_14,
};

const DARK: AppTheme = {
  isDark: true,
  page: HERO_DARK,
  card: HERO_DARK_2,
  cardElevated: HERO_DARK_4,
  border: HERO_DARK_4,
  borderSoft: HERO_DARK_2,
  hero: HERO_DARK_5,
  text: TEXT_ON_DARK,
  textOnDark: TEXT_ON_DARK,
  textSoft: TEXT_ON_DARK_SOFT,
  muted: MUTED_19,
  meta: MUTED_14,
  bannerBg: HERO_DARK_2,
  navBg: HERO_DARK_5,
  navIconActive: AMBER_ACCENT,
  navIconInactive: MUTED_14,
};

export function useAppTheme(): AppTheme {
  const scheme = useColorScheme();
  return scheme === "dark" ? DARK : LIGHT;
}
```

## PR 4 · `shared-components` — adopt the new tokens

This is the biggest PR by file count but the changes are mechanical.
Each file consumes `useAppTheme()` for colors and the new geometry tokens
for radius/spacing. Keep all other props and behavior identical.

### Pattern to apply per component

```tsx
// Before:
import { CARD_BG, BORDER, TEXT_DARK_5 } from "@/lib/design-tokens";
import { FONT_DISPLAY_BOLD } from "@/lib/fonts";

const styles = StyleSheet.create({
  card: {
    backgroundColor: CARD_BG,
    borderColor: BORDER,
    borderRadius: 24,            // ← bare number
    borderWidth: 1,
    padding: 12,
  },
  title: {
    color: TEXT_DARK_5,
    fontFamily: FONT_DISPLAY_BOLD,
  },
});

export function ItemCard(props) {
  return <Pressable style={styles.card}>…</Pressable>;
}

// After:
import { RADIUS_ITEM_AIRY, SHADOW_SOFT } from "@/lib/design-tokens";
import { FONT_DISPLAY_EDITORIAL } from "@/lib/fonts";
import { useAppTheme } from "@/components/use-app-theme";

const styles = StyleSheet.create({
  // Static geometry stays in StyleSheet.create — only color & font become dynamic.
  card: {
    borderRadius: RADIUS_ITEM_AIRY,   // 28 from tokens
    borderWidth: 1,
    padding: 12,
  },
  title: {
    fontFamily: FONT_DISPLAY_EDITORIAL,
  },
});

export function ItemCard(props) {
  const theme = useAppTheme();
  return (
    <Pressable
      style={{
        ...styles.card,
        backgroundColor: theme.card,
        borderColor: theme.border,
        ...SHADOW_SOFT,
      }}
    >
      …
      <Text style={{ ...styles.title, color: theme.text }}>{title}</Text>
    </Pressable>
  );
}
```
