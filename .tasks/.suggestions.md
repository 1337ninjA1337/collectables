# Upgrade Suggestions

- Set `EXPO_PUBLIC_APP_URL` as a build-time environment variable in CI/GitHub Actions so `getAppBaseUrl()` always returns the exact deployed URL instead of inferring it from `window.location.pathname` at runtime. This makes deep links reliable regardless of which page the user is on when they trigger the share flow.
- Wire the new test suite into a GitHub Actions workflow that runs `npm test` on every push and PR. The tests are fast (≈1s locally) and would catch regressions in the pure logic (placeholder color hashing, AI JSON parsing, Cloudinary URL parsing, seed integrity, i18n key coverage) before they reach main. Also fix the pre-existing `tsc --noEmit` errors in `app/profile/[id].tsx` (`activeProfile possibly null`) and add `npx tsc --noEmit` to the same workflow so type regressions are caught too.
- When a private collection is opened via a shared deep link we now auto-register the visitor as a viewer, but there's no "revoke access" UI for the owner yet. Add a list of everyone who currently has access on the share sheet (not just the friends toggle list) so an owner can see link-granted viewers and remove them if needed. This turns the implicit deep-link grant into an auditable permission list and reduces the risk of a leaked link giving indefinite access.
