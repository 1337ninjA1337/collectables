# Upgrade Suggestions

- Add a TTL to the new `viewerProfiles` cache in `SocialProvider`. Today an entry stays in memory for the entire session, so a name/avatar change made elsewhere only shows up after the user signs out and back in. Stamping each cache entry with `cachedAt` and refetching anything older than ~10 minutes inside `ensureProfilesLoaded` would close the staleness window without giving up the dedup wins from this task.

- Cache `node_modules` and the Expo Metro cache (`~/.expo`, `node_modules/.cache`) between CI runs in both `ci.yml` and `release.yml`. The new web-build step adds ~1min per PR push that's mostly Metro re-bundling unchanged dependencies; an `actions/cache` step keyed on `package-lock.json` would cut that close to zero on warm runs without changing semantics.

- Add a small `lib/cloudinary-config.ts` (pure, no react-native import) that reads the three `EXPO_PUBLIC_CLOUDINARY_*` env vars once and exports a single `cloudinaryConfig` object (apiBase, cloudName, uploadPreset). Today `lib/cloudinary.ts` reads `process.env` at module top-level — fine for production but invisible in tests. A pure config module with a `resolveCloudinaryConfig(env)` helper that takes a record makes it trivial to add coverage for the priority rules (override URL wins over cloud name, etc.) and lets future modules share the same parsed config without re-reading env in three places.

- Replace the dual fixed-interval polls landed in `app/chat/[id].tsx` (8s) and `app/chats.tsx` (15s) with a single visibility-aware scheduler that pauses when the tab is hidden / app is backgrounded and resumes (with one immediate refresh) on return. Today both intervals run continuously regardless of focus, which wastes Supabase REST quota on idle tabs and adds avoidable battery drain on mobile. A small `useVisibilityRefresh(refreshFn, intervalMs)` hook in `lib/chat-context.tsx` (listening to `document.visibilitychange` on web and `AppState` change on native) would centralise the cadence and let future cloud-backed screens (premium, marketplace) reuse the same primitive.
