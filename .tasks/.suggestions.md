# Upgrade Suggestions

- Generate `docs/powerbi-connection.md`'s schema-reference table from `lib/analytics-events.ts` at build time (e.g. a `scripts/generate-powerbi-schema-doc.ts` step) so a new event added to the typed union shows up in the BI doc automatically. Today the property-key sub-table is hand-maintained; a future event that's emitted but undocumented becomes an "unknown column" in Power Query.

- Add a Power BI quick-start CLI under `scripts/print-powerbi-conn.ts` that reads the user's `.env` and prints the exact session-pooler connection string + service-role placeholder so engineers don't have to dig through the Supabase dashboard. Useful onboarding helper that pairs nicely with `docs/powerbi-connection.md` step 2.

- Document a free **Metabase** fallback path alongside the Power BI guide for Linux engineers who can't run Power BI Desktop. Same `analytics_events` table, same DAX-equivalent SQL queries; a `docs/metabase-connection.md` companion would close the platform-parity gap without us paying for cross-platform Power BI.

- Add a `docs/powerbi-cost-projection.md` companion that translates "free Power BI Desktop is fine for personal dashboards" into a concrete tipping point ("you'll need Power BI Pro at \$10/user/mo when you want scheduled refresh and >2 viewers"). Today the doc says "manual refresh is usually fine" but doesn't surface *when* to budget for the upgrade.

- Wire a CI-side query that runs the three DAX measures' SQL-equivalent against a tiny seeded `analytics_events` fixture and asserts the expected DAU / funnel / conversion-rate values. Today the DAX is documented but not validated; a one-off `__tests__/powerbi-dax-equivalents.test.ts` translating each measure into raw SQL would catch a typo in the documented formulas before someone wires them into a real dashboard.

- Replace the hand-rolled `timingSafeEqual` in `supabase/functions/analytics-mirror/index.ts` with Deno's `crypto.timingSafeEqual` once a TextEncoder helper is added. Today the JS-level XOR loop is correct but a primitive crypto call is shorter and avoids the per-character UTF-16 quirk on multi-byte secrets.

- Add a `payload-too-large` 413 response to `analytics-mirror` capped at, say, 256KB. Today PostHog could batch hundreds of events into one POST and we'd attempt the insert; bounding the body up front is cheaper than letting Postgres reject a huge insert and is more defensive against a misconfigured PostHog destination.

- Add an `EXPO_PUBLIC_POSTHOG_WEBHOOK_TIMEOUT_MS` knob and respond 504 if the Supabase insert exceeds it. Today a slow/hung Postgres connection would tie up the Edge Function until the platform's hard timeout; a self-imposed 5s ceiling lets PostHog retry sooner with the standard exponential backoff.

Today a malformed event is silently 207'd; cross-tagging into Sentry gives operations a single pane of glass instead of having to grep Edge Function logs.
