# Upgrade Suggestions

- Add a small `lib/cloudinary-config.ts` (pure, no react-native import) that reads the three `EXPO_PUBLIC_CLOUDINARY_*` env vars once and exports a single `cloudinaryConfig` object (apiBase, cloudName, uploadPreset). Today `lib/cloudinary.ts` reads `process.env` at module top-level — fine for production but invisible in tests. A pure config module with a `resolveCloudinaryConfig(env)` helper that takes a record makes it trivial to add coverage for the priority rules (override URL wins over cloud name, etc.) and lets future modules share the same parsed config without re-reading env in three places.
