## Upgrade suggestions

### Chat DB (2026-05-16)

- **Cursor pagination for `fetchMessagesForChat`** ‚Äî it currently `select=*&order=created_at.asc` with no `limit`, so a long-lived conversation pulls the entire history into memory on every hydrate. Add `&limit=50` + a `created_at` cursor (or PostgREST `Range` header) and load older pages on scroll-up. Bounds memory and free-tier egress.

- **Realtime reconnect backfill** ‚Äî while the WebSocket is down, `subscribeToInbox` misses INSERTs and only `refreshFromCloud` (mount-time) closes the gap. On `SUBSCRIBED` after a drop, fetch messages `created_at=gt.<last-known>` per active chat so no message is silently lost during a flaky connection.

- **First-class conversations table** ‚Äî replace the free-text `chat_id` with a `conversations(id uuid, user_a uuid, user_b uuid, UNIQUE(user_a,user_b))` table and a `chat_messages.conversation_id uuid REFERENCES conversations(id)`. A real FK + unique participant pair is stronger than the derived-string CHECK and unlocks per-conversation metadata (muted, archived, last_message_at) without re-deriving from messages.

- **Functional idempotency test with a mocked fetcher** ‚Äî the new ignore-duplicates path is only covered structurally. Add a `__tests__/supabase-chat-idempotency.test.ts` that injects a fake `fetcher` returning `200 []` and asserts `sendMessage` resolves to the synthesized message (not `null`), plus a fake returning a populated row for the happy path.

- **`chat_messages.client_id` + server uuid split** ‚Äî keep the server `id uuid DEFAULT gen_random_uuid()` authoritative and add a separate indexed `client_id uuid` (the composition-time key) with a UNIQUE constraint for idempotency. Decouples "who owns the primary key" from "idempotency key", which is the canonical chat-storage pattern and avoids ever trusting a client-supplied PK.

- **SPA fallback regression guard** ‚Äî Add a CI step that runs `npm run build` then asserts (a) `dist/404.html` contains `location.replace`, (b) `dist/index.html` contains `data-spa-restore`. This catches a future contributor accidentally reverting the post-build script (e.g. swapping back to `cp dist/index.html dist/404.html`) before the change reaches production iOS users.

- **Service worker for instant deep-link load** ‚Äî Even with the sessionStorage redirect, the user sees a flash of the redirect page on iPhone. Registering a service worker (Workbox `NetworkFirst` + `NavigationFallback` to `/collectables/index.html`) would let cached deep links resolve to 200 directly and remove the redirect entirely on repeat visits.

- **Sentry breadcrumb for SPA URL restore** ‚Äî Drop an `addBreadcrumb("spa-redirect", { from, to })` inside the restore script (gated on `__sentry_loaded__`) so Sentry sessions show exactly which deep-link URLs hit the 404 fallback. Useful for ranking which routes deserve a real static export.

- **Static export of common collection IDs** ‚Äî Expo Router could pre-render the top-N most-shared collections at build time (e.g. by reading from Supabase) so their URLs return real 200s without the redirect dance. Reduces flicker and helps SEO for public collections.

- **Migrate baseUrl to a single source** ‚Äî `app.json` declares `expo.experiments.baseUrl`, `lib/url-helpers.ts` infers it at runtime, and `scripts/build-spa-fallback.ts` re-reads it from app.json. A single `lib/base-url.ts` exporting a const + a helper that all three import would eliminate the drift risk if the deployment path ever changes (e.g. moving off the `1337ninja1337.github.io/collectables` prefix).

### Power BI starter template (2026-05-17)

- **`.pbit` open-validation in CI** ‚Äî A hand-built OPC package can be byte-valid yet rejected by Power BI Desktop over a schema nuance. Add a CI step (Windows runner or `pbi-tools`/`Tabular Editor` CLI) that opens `Collectables-Starter.pbit` headless and asserts the model loads. Today only the ZIP structure + JSON shape is tested, not Power BI acceptance.

- **Embed a sample dataset for offline preview** ‚Äî Ship a tiny `analytics_events` seed (a `Table.FromRows` M branch toggled by a `UseSampleData` boolean parameter) so evaluators see DAU/funnel visuals render without a live Supabase connection. Removes the "blank report until you wire credentials" first-run cliff.

- **Generate the `properties` jsonb column expansions** ‚Äî `queries.m` parses `properties` to a record but leaves expansion manual. Drive per-event column expansion from `lib/analytics-events.ts` (`props` arrays) at template-build time so `signup_completed.method`, `listing_created.mode`, etc. arrive as real columns out of the box.

- **Theme the report with the brand palette** ‚Äî Add a `docs/powerbi/collectables-theme.json` Power BI theme (the `#261b14`/`#fff7ef`/`#d89c5b` palette from CLAUDE.md) and reference it via a `resourcePackage` in `Report/Layout` so the starter dashboard matches the app instead of the default Power BI theme.

- **`.pbit` ‚Üî docs parity test** ‚Äî `__tests__/pbit-template.test.ts` checks the `.pbit` vs the text assets, but `docs/powerbi-connection.md` ¬ß5 still hand-copies the same DAX. Add a test asserting every measure body in `measures.dax` appears verbatim in `powerbi-connection.md` ¬ß5 so the three sources (doc, assets, `.pbit`) can never drift.

### analytics_events RLS verification (2026-05-17)

- **Automated RLS probe in CI** ‚Äî Add an opt-in GitHub Actions job (gated on a `SUPABASE_SERVICE_ROLE_KEY` + `SUPABASE_ANON_KEY` secret) that curls `/rest/v1/analytics_events` with the anon key and fails if the response is anything other than 401/empty. Turns the manual checklist into a regression gate that runs on every deploy.

- **`pg_policies` assertion migration** ‚Äî Ship a `supabase/migrations/*_assert_analytics_rls.sql` that `DO $$ ... RAISE EXCEPTION` if `pg_policies` has any row for `analytics_events` or if `has_table_privilege('anon','public.analytics_events','SELECT')`. A self-checking migration makes the lock-down enforceable in the DB itself, not just docs.

- **Generalise the checklist to every RLS table** ‚Äî `marketplace_listings` and `chat_messages` also rely on RLS. Promote the anon/authenticated `SET ROLE` probe into a single `MANUAL-TASKS.md` "RLS verification matrix" table covering all three so a new table can't ship without a documented check.

- **Supabase advisor link** ‚Äî Reference Supabase's built-in Security Advisor (Dashboard ‚Üí Advisors) in the checklist; it flags RLS-disabled / overly-permissive tables automatically and is a zero-effort second line of defence next to the manual SQL.

- **Negative test for the Edge Function path** ‚Äî `analytics-mirror` inserts with the service-role key; add a checklist step (or function test) confirming the function rejects a request whose `x-posthog-webhook-secret` is wrong, so the only write path into the locked table can't be driven by an attacker who discovered the endpoint URL.

### trackEvent rate limiter (2026-05-17)

- **Per-event-name sub-budgets** ‚Äî One global 200/min window lets a single noisy event (e.g. a mis-wired `chat_opened`) starve all others. A `Map<AnalyticsEventName, number[]>` with a smaller per-name cap (say 60/min) plus the global ceiling would isolate a runaway event without losing unrelated funnel signal.

- **Drop-counter breadcrumb** ‚Äî When the limiter rejects an event, increment a counter and, once per window, `addBreadcrumb`/`console.warn` "analytics rate limit hit (N dropped)". Today drops are silent, so a chronically over-budget screen is invisible until the PostHog bill/cap is investigated.

- **Env-tunable cap** ‚Äî Read `EXPO_PUBLIC_ANALYTICS_MAX_EVENTS_PER_MIN` (validated, positive, fallback 200) so QA/load tests can tighten or loosen the window without a code change ‚Äî mirrors the `EXPO_PUBLIC_PROFILE_CACHE_TTL_MS` pattern already in `lib/social-context.tsx`.

- **Shared sliding-window helper** ‚Äî `lib/sentry.ts` and `lib/analytics.ts` now hand-roll the identical `recentEvents.filter(cutoff)` window. Extract `createSlidingWindowLimiter(max, windowMs)` into `lib/rate-limit.ts` and have both consume it; one tested primitive instead of two copies that can drift.

- **Coalesce identical bursts** ‚Äî Instead of hard-dropping over-cap events, debounce-coalesce repeated `(name, props)` pairs within the window into a single event carrying a `count` property. Preserves the signal that "X happened a lot" while still bounding the request volume.
