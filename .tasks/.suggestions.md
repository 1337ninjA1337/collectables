# Upgrade Suggestions

- Codify the missing iOS `infoPlist` block from `APPSTORE-SUBMISSION.md` (NSPhotoLibraryUsageDescription, NSCameraUsageDescription, ITSAppUsesNonExemptEncryption, CFBundleLocalizations, associatedDomains) directly into `app.json` so the next `eas build --platform ios` produces a TestFlight-ready binary without a separate config pass. Today the strings live in the markdown checklist; landing them in `app.json` removes the manual step and lets a structural test enforce that every supported language stays in `CFBundleLocalizations`.

- Add a `scripts/check-appstore-config.ts` validator that's wired into `lint:ci` and asserts `app.json` has a non-empty `expo.ios.bundleIdentifier`, an `expo.ios.icon` path that resolves on disk, and the four required `infoPlist` keys. Today a regression that drops `bundleIdentifier` would only be caught at `eas submit` time; a 30-line pre-flight check catches it locally and in PR CI.

- Generate the App Store Connect privacy nutrition labels from a typed `lib/privacy-manifest.ts` module so the `APPSTORE-SUBMISSION.md` "App Privacy" table and the future iOS 17 `PrivacyInfo.xcprivacy` file stay in sync. Today the table is hand-maintained; a single source of truth that emits both Markdown and the Apple-required plist closes the drift gap before iOS 18+ enforcement.

- Add an `eas-secrets-parity.test.ts` structural test that reads `.github/workflows/deploy.yml` + `APPSTORE-SUBMISSION.md` + `eas.json` and asserts every `EXPO_PUBLIC_*` referenced in one is referenced in the other two. Today the GitHub Actions secrets and the EAS secrets are documented separately — a parity test would catch a future env var added to web deploys but missed in iOS builds.

- Wire a CI matrix job that runs `eas build --platform ios --profile preview --non-interactive --no-wait` on `main` so we always have a fresh internal-testflight build available without a manual trigger. Today the iOS build is purely operator-initiated; an automated build-on-merge job (paid EAS minutes permitting) keeps the App Store track aligned with the GitHub Pages track and surfaces native-only regressions (Hermes / Reanimated worklet failures) within minutes of merge instead of at submission time.
